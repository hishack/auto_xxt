# 学习通题目获取与解密原理分析

## 一、概述

这个油猴脚本用于提取超星学习通平台上的考试/测验题目，并自动解密被混淆的文字内容。学习通使用了**自定义字体加密**技术来防止题目被直接复制，该脚本通过解析字体文件并建立映射关系来还原真实内容。

---

## 二、学习通的加密机制

### 2.1 字体混淆原理

学习通在页面中使用了名为 `font-cxsecret` 的自定义字体，这个字体将：
- 正常的汉字Unicode码位映射到错误的字形
- 导致用户看到的是正确文字，但复制出来是乱码
- 字体文件以 **base64** 格式嵌入在页面的 `<style>` 标签中

### 2.2 加密流程示意

```
真实文字 "答案"
    ↓
显示时使用自定义字体 font-cxsecret
    ↓
Unicode码位 → 错误的字形映射
    ↓
用户看到 "答案"（正确显示）
但复制得到 "㗊㕮"（乱码）
```

## 三、脚本的解密流程

### 3.1 整体流程图

```
1. 加载字体映射表 (table.json)
         ↓
2. 从页面提取 base64 字体文件
         ↓
3. 解析字体文件获取字形数据

         ↓
6. 在映射表中查找对应的真实字符
         ↓
7. 替换并输出解密后的文本
```

### 4.2 解析页面字体文件

**位置**: [jm.user.js:162-203](jm.user.js#L162-L203)

```javascript
function parsePageFont() {
  const styles = document.getElementsByTagName('style');
  let fontBase64 = null;

  // 1. 查找包含 font-cxsecret 的 style 标签
  for (let style of styles) {
    const content = style.textContent;
    if (content.includes('font-cxsecret') && content.includes('base64,')) {
      // 2. 提取 base64 内容
      const match = content.match(/base64,([\w\W]+?)'/);
      if (match && match[1]) {
        fontBase64 = match[1];
        break;
      }
    }
  }

  if (fontBase64) {
    // 3. Base64 解码
    const binary_string = window.atob(fontBase64);
    const len = binary_string.length;
    const bytes = new Uint8Array(len);
    for (let i = 0; i < len; i++) {
      bytes[i] = binary_string.charCodeAt(i);
    }

    // 4. 使用 Typr.js 解析字体文件
    const font = Typr.parse(bytes)[0];
    currentFontData = font;
    fontLoaded = true;
  }
}
```

**关键步骤**：
1. 遍历页面所有 `<style>` 标签
2. 查找包含 `font-cxsecret` 和 `base64,` 的样式定义
3. 使用正则表达式提取 base64 编码的字体数据
4. 将 base64 解码为二进制数据（Uint8Array）
5. 使用 **Typr.js** 库解析 TTF/WOFF 字体文件结构

### 4.3 文本解密核心算法

**位置**: [jm.user.js:215-263](jm.user.js#L215-L263)

```javascript
function decryptText(text) {
  if (!text || !fontHashParams || !currentFontData) return text;

  const md5Fn = getMd5Fn();
  let result = "";

  // 遍历文本的每个字符
  for (let i = 0; i < text.length; i++) {
    const char = text[i];
    const code = char.charCodeAt(0);

    // 1. 获取字符在自定义字体中的字形索引
    const glyphIndex = Typr.U.codeToGlyph(currentFontData, code);

    if (glyphIndex > 0) {
      // 2. 获取字形的路径数据（矢量图形）
      const path = Typr.U.glyphToPath(currentFontData, glyphIndex);

      if (path) {
        const pathStr = JSON.stringify(path);

        // 3. 计算路径的 MD5 哈希值，并截取后8位
        const hash = md5Fn(pathStr).slice(24);

        // 4. 在映射表中查找对应的真实字符
        let match = fontHashParams[hash];

        if (match) {
          // 5. 转换为真实字符
          if (typeof match === 'number') {
            result += String.fromCharCode(match);
          } else {
            result += match;
          }
          continue;
        }
      }
    }

    // 未找到映射，保留原字符
    result += char;
  }
  return result;
}
```

**解密原理详解**：

1. **字形索引查找**：
   - 使用 `Typr.U.codeToGlyph()` 根据 Unicode 码位获取字形索引
   - 如果 `glyphIndex > 0`，说明该字符在自定义字体中有定义

2. **字形路径提取**：
   - 使用 `Typr.U.glyphToPath()` 获取字形的矢量路径数据
   - 路径数据是一个对象，包含了绘制该字形的所有坐标点

3. **MD5 哈希计算**：
   - 将路径对象序列化为 JSON 字符串
   - 计算 MD5 哈希值（32位十六进制）
   - **关键**：取 `slice(24)` 截取后8位作为查找键

4. **映射表查找**：
   - 在预加载的 `fontHashParams` 中查找该哈希值
   - 找到对应的真实 Unicode 码位或字符

5. **字符还原**：
   - 将映射表中的数值转换为字符（`String.fromCharCode()`）
   - 替换原来的乱码字符

## 五、题目提取流程

### 5.1 DOM 结构识别

```javascript
function extractContent() {
  parsePageFont(); // 每次提取前刷新字体解析

  // 1. 查找所有题目块
  const questions = document.querySelectorAll('.TiMu');

  questions.forEach((q, index) => {
    // 2. 提取题目标题
    let titleDiv = q.querySelector('.Zy_TItle .clearfix') ||
                   q.querySelector('.Zy_TItle') ||
                   q.querySelector('.newZy_TItle') ||
                   q.querySelector('.fontLabel');

    let titleText = titleDiv ? titleDiv.innerText.trim() : "未找到题目";
    titleText = decryptText(titleText); // 解密题目

    // 3. 提取选项
    const options = q.querySelectorAll('ul li');
    options.forEach(opt => {
      let optText = opt.innerText.trim();
      optText = decryptText(optText); // 解密选项

      // 判断是否被选中
      const isChecked = opt.querySelector('input:checked') ||
                       opt.querySelector('.ri') ||
                       opt.querySelector('.dui');
    });

    // 4. 提取答案/解析
    const answerDiv = q.querySelector('.newAnswerBx') ||
                     q.querySelector('.answerBx') ||
                     q.querySelector('.lookAnswer');

    if (answerDiv) {
      let answerText = answerDiv.innerText.trim();
      answerText = decryptText(answerText); // 解密答案
    }
  });
}
```

**提取策略**：
- 使用 `.TiMu` 类选择器定位题目块
- 兼容多种题目标题选择器（`.Zy_TItle`、`.newZy_TItle` 等）
- 提取选项列表（`ul li`）
- 识别已选中的选项（`input:checked`、`.ri`、`.dui` 类）
- 提取答案区域（`.newAnswerBx`、`.answerBx`）
- **对所有文本内容调用 `decryptText()` 进行解密**

## 六、关键技术点总结

### 6.1 依赖的外部资源

1. **Typr.js**：
   - 用于解析 TTF/WOFF 字体文件
   - 提供字形索引和路径提取功能
   - 引用：`@require https://scriptcat.org/lib/668/1.0/TyprMd5.js`

2. **MD5 库**：
   - 计算字形路径的哈希值
   - 包含在 TyprMd5.js 中

3. **映射表 (table.json)**：
   - 预先建立的哈希到字符的映射关系
   - 引用：`@resource Table https://www.forestpolice.org/ttf/2.0/table.json`

### 6.2 为什么要 slice(24)？

```javascript
const hash = md5Fn(pathStr).slice(24);
```

- MD5 哈希值是 32 位十六进制字符串
- `slice(24)` 截取第 25-32 位，即**后8位**
- 这是因为映射表中的键值就是使用后8位哈希建立的
- 减少存储空间，8位十六进制已足够区分不同字形

### 6.3 解密的局限性

1. **依赖映射表**：
   - 如果学习通更新字体文件，映射表需要同步更新
   - 映射表不完整会导致部分字符无法解密

2. **字形唯一性**：
   - 假设不同字符的字形路径哈希不会冲突
   - 实际上 MD5 后8位可能存在碰撞风险

3. **性能考虑**：
   - 每个字符都需要计算字形路径和 MD5
   - 对于长文本可能有性能开销

## 七、导出功能

脚本提供了三种导出方式：

1. **预览模式**：在弹窗中显示解密后的文本
2. **导出 TXT**：纯文本格式，包含题目、选项、答案
3. **导出 Word**：HTML 格式的 .doc 文件，带样式

**文件名生成规则**：
```javascript
function getExportFileName(extension) {
  let name = "学习通题目";

  // 尝试从页面标题提取
  const target = document.querySelector(".ceyan_name h3");
  if (target) {
    name = target.innerText.trim();
  }

  const date = new Date();
  const timeStr = `${date.getMonth() + 1}月${date.getDate()}日`;

  return `${name}_${timeStr}.${extension}`;
}
```
